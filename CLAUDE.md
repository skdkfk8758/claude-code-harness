# Claude Code Harness — Project Rules

## Interview Rules

사용자에게 질문할 때 반드시 따를 것:

1. **한 턴 질문 한도** — 의존성 없는 질문 최대 2개, 의존성 있으면 1개씩 순차
2. **추천은 답변 후에** — 선택지 제시 시 추천을 먼저 하지 않음. 사용자 답변 후 의견 제시
3. **시나리오 역검증** — 추상적 규칙 확정 후 구체적 예시 3-5개로 역검증 수행
4. **맥락 수집 우선** — 질문 전에 코드베이스/기존 데이터를 먼저 스캔. 이미 알 수 있는 것을 묻지 않음
5. **선택지 공정성** — 모든 옵션에 동등한 톤으로 장단점 기술. 특정 옵션에 부정적 뉘앙스 편중 금지
6. **완료 기준** — 인터뷰 종료 시 결정 사항 요약 테이블 제시. 명시적 확인 후 다음 단계 진행

## Git & Release 언어 규칙

커밋 메시지, PR 제목/본문, 릴리즈 노트 등 **Git 관련 산출물은 모두 한글로 작성**한다.

- **커밋 메시지**: type(scope)은 영문 유지, 설명은 한글
  - 예: `feat(workflow): 자동 브랜치 생성 기능 추가`
  - 예: `fix(plan-reviewer): steel-man 단계 누락 수정`
- **PR 제목/본문**: 한글로 작성. 기술 용어(tool name, API name 등)는 원문 유지
- **릴리즈 노트**: 한글로 작성. 섹션 헤더, 변경 내역, 설명 모두 한글
- **Co-Authored-By**: 영문 유지 (Git 규격)
