# DOT Combo: Systematic Debugging

## Trigger
User reports a bug, error, or unexpected behavior.

## Rules
1. Reproduce: confirm the issue with a concrete test case or steps
2. Hypothesize: list top 3 most likely root causes
3. Narrow: use binary search or elimination to isolate the cause
4. Fix: apply the minimal change that resolves the root cause
5. Verify: confirm the fix resolves the issue without regressions

## Routing
- Mode: code (DOT enabled)
- Source: superpowers/systematic-debugging (migrated)
- Fallback: direct investigation without structured phases
