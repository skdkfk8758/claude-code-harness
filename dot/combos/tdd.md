# DOT Combo: TDD (Test-Driven Development)

## Trigger
User requests implementation of a new feature or function with test coverage.

## Rules
1. Write failing test first (Red)
2. Write minimal implementation to pass (Green)
3. Refactor while keeping tests green (Refactor)
4. Each cycle should be small and focused on a single behavior
5. Report cycle progress: Red -> Green -> Refactor

## Routing
- Mode: code (DOT enabled)
- Source: superpowers/tdd (migrated)
- Fallback: write implementation first, then tests (standard flow)
